Spent the last two weeks developing the signal processing pipeline. Raw sensor data is far too noisy to drive lighting directly — the result is jittery, nervous light that feels more like a malfunction than an artistic response.

The solution involves three stages:

  1. Median filtering — a rolling window of 5 samples removes outlier spikes without introducing lag.
  2. Exponential moving average — smooths the filtered signal with a configurable time constant. For temperature, a 30-second time constant works well. For proximity, 200 milliseconds is sufficient.
  3. Threshold gating — small fluctuations below a configurable threshold are suppressed entirely, so the light only responds to meaningful changes.

The key insight is that different environmental parameters require very different temporal treatment. Temperature changes slowly and should drive slow, gradual light transitions. Sound is fast and transient but should be averaged over several seconds to capture the ambient character rather than individual sounds. Proximity sits in between.

I wrote a small Python library (envsmooth) on the Crestboard that handles all of this. Each sensor channel gets its own filter chain with independently configurable parameters.
